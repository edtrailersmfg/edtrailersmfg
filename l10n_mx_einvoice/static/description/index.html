Facturación Electrónica 4.0

Este modulo nos permite trabajar con la facturación electrónica CFDI 4.0

CONFIGURACIÓN

Compañia

Debemos ingresar lo datos dentro de la Compañia:
  • Nombre o Razón Social
  • Dirección
  • RFC
  • Regimen Fiscal


Diarios

Debemos ingresar lo datos del Certificado de Sello Digital Emisor dentro de cada Diario, tanto de facturación como para pagos (bancos, efectivo, etc..)
  • Activamos el Campo Usar para CFDIS


Se habilitara la pestaña Config CFDI y vamos a ingresar los siguientes datos:
  • Compañia Emisora
  • Dirección de Emisión
  • Archivo CER
  • Archivo KEY
  • Contraseña del certificado
Los demas datos se calculan en automatico.


Clientes

Debemos ingresar lo datos dentro de cada Cliente:
  • Activar Compañia o Persona Fisica
  • Dirección (Codigo Postal Obligatorio)
  • RFC
  • Regimen Fiscal


Impuestos

Debemos ingresar lo datos dentro de cada Impuesto:
  • Tasa o Cuota
  • Clave SAT
  • Importe o Porcentaje


Productos

Debemos ingresar lo datos dentro de cada Producto:
  • Impuestos (previamente configurados)
  • Referencia Interna
  • Clave SAT
  • Tipo de Identificación (Incluida en el Nodo NoIdentificación de cada concepto del XML)


Unidades de Medida

Debemos ingresar lo datos dentro de cada Unidad de Medida
  • Clave SAT


EXTRAS

Categorias

La Clave SAT puede existir a nivel Categoria:
  • Clave SAT
Si la categoria no tiene Clave tomara la definida en el producto.


Terminos de Pago

El Metodo de Pago se puede automatizar ingresando el dato en cada uno de los Terminos de la Base:
  • Metodo de Pago
Al seleccionarse en la factura un termino de pago se ingresara el metodo asociado a este.


FACTURACIÓN

Facturas

Debemos ingresar la información requerida por el SAT:
  • Cliente (previamente configurado)
  • Metodo de Pago
  • Uso de CFDI
  • Forma de Pago
  • Lineas de Factura.
La información sera enviada al SAT.


Si todo es correcto tendremos el XML devuelto por el PAC (Adjunto en la Factura):


Si configuramos un servidor de correo saliente, la factura sera enviada automaticamente junto a su representación impresa:
PAGOS

Pagos 2.0

Si el diario del Pago esta configurado previamente para el Timbrado el Pago 2.0 es generado automaticamente:
  • Usar para CFDI


Abrimos el Pago creado desde la Factura y activamos e ingresamos la información:
  • Generar CFDI
  • Tipo de Complemento (Contabilidad Electronica)
  • Metodo de Pago
  • Forma de Pago
  • Fecha efectiva de Pago (Enviado al XML)
  • Sin datos Bancarios (esta opció requiere que se tenga configurada correctamente la cuenta de banco de la compañia emisora y la cuenta bancaria en el cliente).
